> **用途**：防范「计划写了、清单有了，但实际没执行」——针对精力分散、临时任务挤占、探索型工作无边界等常见失败模式。
>
> **定位**：执行期守则，配合 [[时间任务管理统一工作流]]、[[番茄执行法]]、[[ABC + 4D工作法判断优先级]] 使用。**不替代** GTD 整理与 4D 筛选，只保护「今日执行」不被无声替换。
>
> **使用时机**：每天打开 [[全局任务池]]「今日执行」前读一遍；中断发生时查「30 秒清单」；晚上 [[tasks/日志/README|日志]] 复盘时对照「每日成功标准」。

---

## 一、问题诊断：你常见的三种分散

| 类型 | 表现 | 典型例子 |
|---|---|---|
| **未登记就开工** | 临时事项直接占用时间，计划被替换却无处记录 | 搭 Obsidian、写决策文书、配置工具 |
| **深度工作堆叠** | 一天排多个需整块专注的任务 | 决策案例 + JD 调研 + 阅读 + 笔记同日硬塞 |
| **探索型无边界** | 「再查一点、再优化一下」没有停手点 | 搜 Agent 资料、改模板、优化工作流 |

> 根因往往不是「不会计划」，而是执行期缺少 **登记、限并发、保护时段** 三道闸。

---

## 二、三条硬规则（每日必守）

### 规则 A：临时任务先登记，再决定做不做

**触发**：任何预计 **>15 分钟** 的事想立刻动手时。

**动作**（约 1 分钟）：

1. 一行写入 [[收件箱]]
2. [[ABC + 4D工作法判断优先级|4D]] 判断：**今天做 / 延后 / 删除**
3. 若今天做 → 写入 [[全局任务池]]「今日执行」，并 **顺延或删除** 一条原计划

**禁止**：未登记就占用 14:30–17:00 深度块或 14:30 第一个职业番茄。

---

### 规则 B：每天最多 1 主 + 1 副

「今日执行」里：

| 槽位 | 数量 | 放什么 |
|---|---|---|
| **主块** | 1 个 | 当天必须推进的 A 类（如 W1 JD 调研、作品集） |
| **副块** | 1 个 | B 类或轻量整理（如阅读、笔记） |

其余任务 **Defer 到明天**。备选区只放「主副都完成才有余力」的事。

**自检**：今日执行若超过 2 个时间块任务，说明又在堆叠，需删减。

---

### 规则 C：14:30 第一个职业番茄给「主块」

打开 [[全局任务池]] 后：

1. **09:00–09:30** 只做 4D 筛选，写入「今日执行」**1 主 + 1 副**
2. **09:30–11:00** 只做轻量任务（B 类 / 预习 / 短整理），**不启动深度主块番茄**
3. **14:30 点 ▶ 启动第一个职业番茄** → 主块
4. 禁止用「先整理环境 / 先优化系统」代替 14:30 真实启动——那是常见的假启动

---

## 三、时段保护（按个人作息）

固定 **不可侵占** 块（与职业决策文书一致）：

| 时段 | 用途 |
|---|---|
| 08:00–09:00 | 早餐 + **检查小孩餐食并计划采购**（家庭 A 类；零职业深度工作） |
| **09:00–09:30** | **4D 筛选** → 「今日执行」填 **1 主 + 1 副** |
| **09:30–11:00** | **轻量任务**（B 类 / 预习 / 短整理；零职业深度工作） |
| 11:00–13:00 | 做饭陪餐，**零工作** |
| 13:00–14:00 | 饭后 / 家庭，**零工作** |
| **14:00–14:30** | **午睡半小时**（零工作） |
| **14:30–17:00** | **唯一深度工作主块** → 只放「主块」任务 |
| 17:00–19:30 | 做饭陪餐，**零工作** |
| **22:00–23:00** | **副块** / 阅读 / 轻整理 |

**原则**：A 类深度工作优先锁进 14:30–17:00；临时插入只能进收件箱或换到明天，不得无声占用主块。09:00–11:00 可 4D / 轻量任务，**不算**职业主块。

## 四、中断处理：30 秒清单

内部中断（想查资料、改模板、优化 Obsidian）或外部打断时：

```
停 → 不切换窗口
记 → 收件箱一行
回 → 回到当前番茄
```

番茄内 **只记录、不切换**；中断事项等当前番茄结束再 4D。

---

## 五、探索型工作预算

以下属于 **探索型**（易无限膨胀）：

- 系统 / 工具 / 模板搭建与优化
- 泛搜索、泛收藏教程
- 流程方法论重写（非当日必要部分）

**限额**：每周最多 **2 个番茄** 用于探索型工作。超出必须 Defer，并自问：

> 这是在推进 offer / 阅读 / 家庭 A 类，还是在用忙碌感逃避难任务？

## 六、每日成功标准（窄定义）

| 结果 | 定义 |
|---|---|
| **成功** | 今日 **主块** 完成 |
| **加分** | 副块也完成 |
| **正常延后** | 主块未完成但有登记原因 + 已顺延全局任务池 |

**禁止**用「上午没做 = 全天失败」否定自己——失败感会诱发「计划破了，再分散也无所谓」。

## 七、每日动作清单

| 时刻 | 动作 | 在哪里 |
|---|---|---|
| 08:00–09:00 | 早餐 + 检查小孩餐食、计划采购 | — |
| 09:00–09:30 | 4D 筛选，「今日执行」只填 **1 主 + 1 副** | [[全局任务池]] |
| 09:30–11:00 | 轻量任务（B 类 / 预习 / 短整理） | 同上 |
| 14:00–14:30 | 午睡半小时 | — |
| 14:30 | 第一个 ▶ 职业番茄启动 **主块** | 「今日执行」Day Planner |
| 22:00–23:00 | **副块**（阅读 / 笔记 / 轻整理） | 同上 |
| 全天 | 中断 → **30 秒清单** | [[收件箱]] |
| 23:00 后 ~10 分钟 | 只写总结；任务状态已在白天于全局任务池更新 | [[tasks/日志/README|日志]] |

**白天只盯一个屏**：[[全局任务池]]「今日执行」。每日节奏详见 [[决策与执行速查]] 第三节。

## 八、晚上复盘三问（写进日志即可）

1. **主块完成了吗？** 若否 → 登记问题 / 估时问题 / 堆叠问题？
2. **有没有未登记就开工的事？** 若有 → 明天启用规则 A。
3. **明天主块是什么？** 备忘即可；明早 4D 时正式写入「今日执行」。
