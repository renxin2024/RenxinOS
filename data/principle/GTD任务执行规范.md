# 执行流程

1. 收集GTD项目 收集到一个 Obsidian [[收件箱]] 中
    
2. 处理GTD项目
    
    1. 这个任务是什么？想法？要做的具体的事情？忘不掉的事情？
        
    2. 我需要做什么具体的事情？
        
    3. 是否可以执行呢？
        
        1. 不能执行的丢弃或者存档
            
        2. 可以执行的进入下一步行动
            
3. 整理GTD项目
    
    1. 已经定好的日程 则 分发到日程清单
        
    2. 暂时依赖别人而自己做不了的进入等待清单
        
    3. 想做但当前不重要或没时间的进入 将来/也许
        
    4. 复杂任务需要进一步拆分的进入 项目清单
        
4. 回顾
    
    1. 查看 将来/也许列表
        
    2. 使用 ABC法排列优先级
        
    3. 检查“等待中”和“将来/也许”列表
        
5. 执行
    
    1. 每天清晨工作之前查看GTD清单，确定今天要执行的优先任务 使用4D法
        
    2. 安排每天要完成的任务时使用
        
    3. 选择三个番茄任务，估算番茄数，加入任务列表，每天最多只有20个番茄
        
    4. 番茄执行 专注25分钟，休息5分钟。要特别重视每天长休息期间注意力分散导致时间管理失控的问题
        
    5. 日回顾，每天结束工作前
        
        1. 看看自己完成了哪些任务
            
        2. 哪些任务需要调整
            
        3. 番茄钟执行情况
            
    

# 各步骤执行方法

| GTD任务     | 执行时机                      | 执行方法                                                                                                                                                                                                                                    |
| --------- | ------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 收集        | 任何时间                      | 任何想法都可以收集到 Obsidian 的收件箱笔记中                                                                                                                                           |
| 处理        | 每天结束工作前<br><br>或者当前无待执行任务 | 1. 这是什么？<br>    <br>2. 我需要做什么？<br>    <br>3. 是否可执行？<br>    <br>4. 是否太模糊需要进一步具体化拆分？需要拆解的进入GTD项目笔记<br>    <br>5. 将收集笔记中的数据转为清单中的任务                                                                                                        |
| 整理        | 每天结束工作前                   | 将处理完的任务加入到不同的清单中 用ABC优先级法列全局任务池，使用smart原则描述目标<br><br>1. 日程表，已经有固定时间安排的<br>    <br>2. 下一步行动清单，已经明确可执行的待安排时间执行<br>    <br>3. 等待中，依赖别人实施的<br>    <br>4. 将来/也许，想做但不是现在要做的事<br>    <br>5. 项目列表，复杂任务需要进一步明确拆分的                                 |
| 启动今日任务    | 每天清晨8点                    | 1. 查看 GTD 清单：确定今天的优先任务<br>    <br>2. 使用4D法列出执行计划<br>    <br>3. 选择 3~5 个番茄任务，估算所需番茄数<br>    <br>4. 每天时间情况 20个番茄<br>    <br>    1. 清晨 7:30 ～9:00 4个🍅<br>        <br>    2. 上午 9:30 ～12:00 5个🍅<br>        <br>    3. 下午13:30～19:00 11个🍅 |
| 执行（番茄工作法） | 工作时间、家庭时间                 | 1. 启动番茄钟，专注25分钟<br>    <br>2. 被中断则记录中断原因<br>    <br>3. 成功则记录做了什么                                                                                                                                                                        |
| 日回顾       | 每天结束工作前                   | 1. 完成了哪些任务<br>    <br>2. 哪些任务没完成是否需要调整<br>    <br>3. 番茄钟执行情况记录<br>    <br>4. 明天最重要的三件事                                                                                                                                                  |
| 周回顾       | 每周六下午                     | 1. 看看哪些任务还没处理<br>    <br>2. 调整任务优先级：使用ABC优先级法<br>    <br>3. 检查等待中和将来/也许列表                                                                                                                                                               |
| 睡眠        | 每天晚上<br><br>每天中午          | 1. 晚上睡足7小时<br>    <br>2. 中午睡足半小时                                                                                                                                                                                                        |
