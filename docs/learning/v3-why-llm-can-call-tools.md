# 为什么大模型能调用工具了？—— 第一性原理拆解

> 问题来源：用户对 V3 手写 Agent 的底层疑问
> 关联：`docs/learning/v3-raw-react-agent.md`、`src/agent_raw/tools.py`

## 1. 核心答案（一句话）

大模型没有"突然学会调用工具"。它和以前一样，只知道**输入文本 → 输出文本**。能调用工具是因为我们教了它一种新的输出格式（JSON），然后在调用方解析这个 JSON 去执行真实函数。

## 2. 不变的东西：模型只是一个文本补全器

无论 GPT-3 还是 GPT-4，底层数学没变：

```
给定 token 序列 [t1, t2, ..., tn]
→ 预测下一个 token P(t_{n+1})
→ 采样一个 token，追加到序列
→ 继续预测下一个
```

它不知道"我在调用工具"，不知道"这是 JSON"，不知道"search_notes 是一个函数"。它只是在生成一串它认为概率最高的字符。

## 3. 变化的东西：输出格式约定

所谓"模型返回工具调用"，还原到 token 层面：

```
模型输出的 token 序列：
{ " n a m e " :   " s e a r c h _ n o t e s " ,   " a r g u m e n t s " :   { " q u e r y " :   " G T D   c h e c k b o x " } }
```

这和模型输出"你好，答案是……"没有任何区别——都是文本。

## 4. 怎么让模型输出这种格式

### 4.1 Fine-tuning（微调）

拿海量对子训练模型：

```
输入（用户问）：GTD 任务的 checkbox 写在哪？
输出（模型要生成的）：{"name":"search_notes","arguments":{"query":"GTD checkbox"}}
```

经过足够多这类样本训练后，模型学到的模式是："当问题问我知识库里的内容时，我应该输出一段 JSON 让调用方去查，而不是自己编答案。"

类比：你背了乘法口诀后看到 3×5 会条件反射输出 15。模型是背了大量"问知识库问题 → 输出查笔记的 JSON"的对子。

### 4.2 System Prompt 注入

在对话开头告诉模型：

```
你有以下工具可用：
- search_notes(query): 搜索个人笔记
- search_tasks(query): 搜索今日任务

需要使用时，按以下 JSON 格式输出：
{"name": "<工具名>", "arguments": {"query": "<搜索词>"}}

不需要使用时，直接回答用户问题。
```

模型把这段指令纳入上下文后，文本补全的方向就被引导了。本质上是在和模型做"格式约定"——类比你在团队里约定好 PR 描述必须按模板写，大家就会照着做。

## 5. 谁在真正"调用工具"？—— 不是模型，是调用方

这是最容易被误解的地方。完整流程：

```
1. 用户问："GTD checkbox 写在哪？"
2. 调用方把问题 + system prompt 发给模型
3. 模型输出一段文本：{"name":"search_notes","arguments":{"query":"GTD checkbox"}}
4. 调用方（你的代码）检查这段文本 → 发现是 tool_call 格式
5. 调用方解析 JSON → 调用真实的 search_notes("GTD checkbox") 函数
6. 调用方把函数返回值注入下一轮对话 → 发给模型
7. 模型基于工具返回的结果生成最终答案
```

**模型只做了第 3 步和第 7 步——生成文本。真正执行函数、读写文件、查询数据库的，永远是调用方。**

类比你熟悉的 Java：
- 模型 = 序列化器，输出"请求对象"的 JSON
- 调用方 = RPC 客户端，反序列化 JSON → 调 RPC → 把返回值塞回上下文
- 工具函数 = RPC 服务端的实际实现

## 7. 面试讲法

**30 秒版**：模型从没"学会调用工具"——它只是被训练成在需要时输出 JSON 格式的工具调用文本。真正执行工具的是调用方代码，模型只负责生成"请求内容"和基于工具返回结果的"最终答案"。

**2 分钟版**：
1. 不变：模型始终是文本补全器，输入 token 序列 → 预测下一个 token
2. 变化：通过 fine-tuning + system prompt，模型学会了一种新的"说话方式"——把工具调用请求以 JSON 格式输出
3. 关键：模型不执行工具。调用方解析模型输出的 JSON，执行真实函数，把返回值注入下一轮对话
4. 映射到你项目：V3 `parse_tool_call()` 就是调用方的解析层；V4 OpenAI SDK 内置 tool_call / MCP `tools/call` 是同一件事的框架化版本
