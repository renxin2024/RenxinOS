# 主流 Agent 模型、评估维度与训练三阶段

> 问题来源：用户从"指令遵循"的追问——哪些模型擅长 Agent？怎么选？预训练/后训练/微调是什么？
> 关联：`docs/learning/v3-why-llm-can-call-tools.md`、`docs/learning/v3-instruction-following-and-agent-model.md`
> ⚠️ 模型名称时效性敏感，以下写于 2026-06-23，请以 BFCL / MT-Bench 最新榜单为准。

## 1. 主流 Agent 开发模型概览

**方法论优先于具体名称**：模型换代极快（3-6 个月一代），记原则比记名称重要。选 Agent 模型的核心原则：

- **首选有原生 tool calling / function calling API 的模型**（而非纯文本模型靠 prompt 硬撑）
- **中文场景优先国内模型**（Qwen 系列、DeepSeek 系列）——它们对中文指令、中文工具名的遵循更好
- **面试展示用最好的模型跑一轮对比**——证明你有"模型选型"的判断力

### 如何自行获取最新模型列表

1. **BFCL 榜单**（Berkeley Function Calling Leaderboard）：搜索 "BFCL leaderboard gorilla berkeley" 即可找到。这是 tool calling 准确率的权威排名
2. **LMSYS Chatbot Arena**：搜索 "lmsys chatbot arena leaderboard" 找到。人类盲评多轮对话排名，含 tool use 子榜
3. **各厂商官网 / 控制台模型列表页**：OpenAI、Anthropic、阿里百炼、DeepSeek 等厂商会在模型发布页标注是否支持 tool calling
4. **Bluesky / X 关注**：@lmsysorg（Arena 更新）、@_akhaliq（Arxiv Daily）

> ⚠️ 以上链接请自行搜索确认具体地址。榜单 URL 可能随版本更新变化，搜索引擎比硬编码链接可靠。

### 对 RenxinOS 的建议

- **当前 V3 练手**：任何支持 function calling 的模型都可以。你用的是百炼上的某个模型，在 V3 手写阶段完全够用
- **V4/V5 展示**：建议用当时 tool calling 榜单前 3 的模型跑完整 Agent 链路
- **对比实验（加分项）**：至少跑 2 个不同厂商的模型，记录 tool calling 准确率差异——面试时这就是"模型选型经验"

## 2. 评估模型是否适合 Agent 的五个维度

| 维度 | 测什么 | 怎么测 | 为什么重要 |
|---|---|---|---|
| **Tool Calling 准确率** | 能否正确选择工具、填对参数 | BFCL 榜单 | 选错工具整个链都断 |
| **格式遵循** | 能否稳定输出合法 JSON | 拿 100 条实际 query 统计格式正确率 | `parse_tool_call()` 不炸 |
| **多轮状态保持** | 10 轮后还记得上下文 | MT-Bench 多轮分 | 长 Agent 链不丢失信息 |
| **推理链质量** | 不确定时是否会多查一步 | HotpotQA / MuSiQue | Agent 核心价值——"我需要更多信息"的判断力 |
| **幻觉控制** | 不知道时说不说"不知道" | SimpleQA / HaluBench | 编造比沉默伤害大得多 |

### 你可以用 V3 项目自己做模型评测

V2 的 16 题评测集 + failure 分类可以扩展：

1. **Tool Calling**：用 `search_notes` + `search_tasks` 场景，统计模型选择正确工具的比例
2. **格式遵循**：统计 `parse_tool_call()` 三种结果（JSON 成功 / 回退文本 / 彻底失败）的比例
3. **多轮保持**：同一个 Agent 跑 5 个连续问题，看第 5 个是否还记得第 1 个的上下文
4. **推理链**：给多跳问题（"今天的任务和哪条原则冲突？"），看模型会不会分步查
5. **幻觉控制**：给不在笔记里的问题（"我的密码是多少？"），看能否拒绝回答

## 3. 预训练 / 后训练 / 微调

### 3.1 一句话区分

| 阶段 | 一句话 | 类比 |
|---|---|---|
| **预训练** | 海量文本随便读，学会"语言规律" | 小孩从 0 到 3 岁听大人说话，学会组词造句 |
| **后训练** | 让模型懂规矩——什么该说、按什么格式说 | 上小学了，老师教"回答问题要先举手，不许骂人" |
| **微调** | 针对某个具体任务再练一轮 | 上大学选专业——计算机的天天写代码，医学的天天看病 |

### 3.2 预训练（Pre-training）

**做什么**：把互联网上几乎所有文本扔给模型做"填空题"——遮住下一个词，让它猜。

**底层原理**：

```
输入：[GTD] [的] [核心] [?]   ← 遮住 "是"
模型猜：下一个词 80% 概率是 "是"，10% 是 "就是"，5% 是 "指"...
正确答案：是 → 模型调整参数，下次更准
```

做完这步的模型叫 **Base Model**——会造句、懂逻辑、有常识，但不会"对话"。

**想要的效果**：模型懂得什么是语言。

### 3.3 后训练（Post-training）

Base Model 虽然会说话，但你问"你是谁"，它可能补全成"你是谁不重要，重要的是……"——只被训练做文本补全，没被训练回答。

后训练两个核心步骤：

**SFT（监督微调）**：拿大量"人类写的优质对话"给模型看，让它学"被问了该怎么答"。

**RLHF（人类反馈强化学习）**：同一个问题生成 4 个回答 → 人类排 1234 → 训练打分模型 → 反过来训练主模型。

**想要的效果**：模型学会对话——有用、安全、格式正确、该拒绝时拒绝。

**关键**：tool calling 的格式训练也发生在后训练阶段。预训练模型从没见过 JSON 格式工具调用。后训练时喂工具调用对子，让模型学会"当用户问这类问题时，输出 JSON 而不是直接回答"。

### 3.4 微调（Domain Adaptation）

模型已经会调工具了，但不知道你项目里有叫 `search_tasks` 的工具。微调拿你自己的数据再训一轮：

```
输入："今天主块是什么？"
期望输出：{"name":"search_tasks","arguments":{"query":"今日主块"}}
```

| 方法 | 怎么做 | 适用场景 |
|---|---|---|
| Full Fine-Tuning | 训练全部参数 | 大量数据 + 多 GPU |
| LoRA | 只训练小的"补丁参数"，冻结主模型 | 几百条数据、消费级 GPU 跑 |

**想要的效果**：模型更熟悉你的领域——工具名不叫错、参数不填错。

### 3.5 三阶段在 RenxinOS 中的体现

| 阶段 | 谁做的 | 你用什么结果 |
|---|---|---|
| 预训练 | 模型厂商 | 模型会理解中文、会推理 |
| 后训练 | 模型厂商 | 模型学会输出 JSON tool_call——你 V3 `parse_tool_call()` 解析到的正是这个阶段的产物 |
| 微调 | （可选，V5 加分项） | 用 LoRA 微调一个模型到你的工具集上，显著提高工具选择准确率 |

## 4. 如何获取最新模型信息（保持笔记不过时）

- **BFCL Leaderboard**：https://gorilla.cs.berkeley.edu/leaderboard.html — tool calling 专榜
- **LMSYS Chatbot Arena**：https://chat.lmsys.org — 人类盲评排名
- **各厂商 Model Card**：模型发布页通常标注了 tool use / function calling 支持情况
- **简单原则**：榜单上前 5 且标注"function calling / tool use"的，就是当前最佳 Agent 模型

## 5. 面试讲法

**30 秒版**：选 Agent 模型看 Tool Calling 准确率、格式遵循、多轮保持、推理链质量、幻觉控制五个维度，不看名气看榜单。预训练学语言、后训练学对话和 tool calling、微调适配你的具体场景。

**2 分钟版**：
1. 模型不记具体名称，记原则：有原生 tool calling API + 看 BFCL 榜单 + 中文用国产模型
2. 训练三阶段本质：预训练 = 学语法；后训练 = 学对话 + tool calling（Agent 的关键步骤就在这）；微调 = 适配你的领域
3. 你的项目映射：V3 手写跑通了解析层；V4 如果换 BFCL 前 3 模型跑 MCP，格式遵循更好
4. 加分项：V5 用 LoRA 微调模型到自己的工具集上，面试能讲完整的训练管线认知
